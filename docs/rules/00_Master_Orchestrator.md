# NebulaOS -- Master Orchestrator Model

## Purpose

Define how multiple AI agents collaborate to design and build NebulaOS.

NebulaOS is: - 100% Open Source - Sovereign Cloud OS - Multi-tenant -
IAM equivalent to AWS IAM (open source) - IaC-first (Terraform, etc.) -
Simple UI like DigitalOcean / Proxmox - Advanced mode for engineers -
Domain, reverse proxy & certificate automation included

------------------------------------------------------------------------

## Execution Phases

### Phase 1 -- Sequential (Foundational)

1.  Vision Architect
2.  Cloud Architect
3.  IAM Architect
4.  Governance Architect

Each output feeds the next.

### Phase 2 -- Parallel (After foundation defined)

-   DevOps Orchestrator
-   UX Architect
-   Network & Edge Architect

### Phase 3 -- Consolidation

-   MVP Strategist

------------------------------------------------------------------------

## Orchestration Rule

Each agent must: - Respect previous architectural decisions - Avoid
vendor lock-in - Preserve modularity - Maintain sovereign-first
philosophy
